BOL Webmail - PostMessage Command Injection PoC

Vulnerability: window.addEventListener("message", handler) with NO origin check

45 commands accessible from ANY origin via postMessage

Step 1: Open target webmail

Step 2: Send commands (no origin check!)

Step 3: Destructive actions

Technical Details

The BOL Webmail (bmail.uol.com.br) has a postMessage listener:

  window.addEventListener("message", function(a) {
    var b = window.windowCmds[a.data.cmd];
    b && b(a);  // NO origin check!
  }, false);

This accepts commands from ANY origin. 45 commands available including:
- goToPath: Navigate to arbitrary routes
- exportConfigsService: Exfiltrate user configuration
- saveDraftFromPopup: Inject draft emails
- doTrashEmailsService_emails: Delete emails
- doSpamEmailsService_emails: Mark as spam
- doMoveEmailsService_emails: Move emails

All postMessage calls in the app use target origin "*".
